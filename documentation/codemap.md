## Navigation notes

- Use `architecture.md` for subsystem behavior and design decisions.
- Use `roadmap.md` for known bugs, limitations, and what's next.
- Use `development_progress.md` for implementation status per subsystem.
- Use `output_contract.md` for the stable generated-output format spec.
- Use `release_checklist.md` before tagging a release.

## Key entry points

| Goal | Start here |
|------|-----------|
| Run the CLI | `DevDocsDownloader.py` → `doc_ingest/cli.py` |
| Understand source adapters | `doc_ingest/sources/registry.py`, `devdocs.py`, `mdn.py`, `dash.py` |
| Understand compilation | `doc_ingest/compiler.py` |
| Understand desktop API | `doc_ingest/desktop_backend.py` |
| Understand WinUI shell | `desktop/DevDocsDownloader.Desktop/App.xaml.cs`, `MainWindow.xaml.cs`, `ViewModels/MainWindowViewModel.cs` |
| Add a source plugin | Implement `DocumentationSource` protocol in `doc_ingest/sources/base.py`, expose via `devdocsdownloader.sources` entry point |
| Run tests | `python -m pytest -q` |
| Freeze backend | `python scripts/build_desktop_backend.py --clean` |
| Build icon | `python scripts/generate_icon.py` |
